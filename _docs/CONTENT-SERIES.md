# Red Pill Wiki - Flagship Public Content Series

Private planning document for public distribution. Kept under `_docs/`, which is excluded from Quartz publishing by `quartz.config.ts`.

## Editorial Rules For All Series

- Vietnamese-first. English keywords only when they sharpen the frame.
- Strong hook, clean claim layer. Do not turn symbol or synthesis into fact.
- Every health, finance, science, and politics post needs a caution line.
- No fake citations. Use only sources already inside the vault, public primary sources, or explicit "source needed" placeholders.
- Each flagship should drive readers back to a small reading path, not a single isolated article.
- Public copy can be sharp; production notes must stay disciplined.

## Priority Scale

- **P0:** launch series; strongest vault spine and broadest audience fit.
- **P1:** publish in first month after P0 starts.
- **P2:** prepare, source-harden, or serialize after stronger audience baseline.

# Repurposing Workflow

## 1. Start From The Vault Article

For each flagship, extract:

- one thesis sentence,
- 5-7 clean claims,
- 3 caution labels,
- 5 related vault nodes,
- 3 reader actions or questions.

Do not start from a viral hook. Start from the map, then sharpen the hook.

## 2. Build The Anchor Package

Each series gets:

- one X/Twitter anchor thread,
- one 45-60 second short video,
- one 1,200-2,000 word newsletter,
- three quote cards or text posts,
- one "common mistake" post,
- one reading-path post.

## 3. Claim-Layer Pass

Before publishing, tag each claim internally:

- **Fact/documentable:** needs source or obvious public record.
- **Pattern/systems:** phrase as pattern reading.
- **Symbol/myth:** phrase as symbolic reading.
- **Speculative synthesis:** phrase as model/hypothesis, not proof.

If a post has health, finance, law/policy, current events, or specific factual allegations, add a source/caution pass before scheduling.

## 4. Platform Adaptation

- **X/Twitter:** compress to one idea per post; use numbered structure; end with reading path.
- **Short video:** one hook, one visual metaphor, one caution, one action.
- **Newsletter:** include nuance, caveats, source categories, and internal vault path.
- **Website:** link back to the full vault article or gateway page.

## 5. Feedback Loop

After 7 days, log:

- best hook by saves/shares,
- most misunderstood claim,
- strongest objection,
- source requests,
- next vault note readers ask for.

Turn misunderstandings into "How to read this correctly" posts. Turn source requests into future source-hardening tasks, not fake citation filler.
