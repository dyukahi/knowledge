## Page targeting notes

### Quick wins

1. **Descriptions are now present across public markdown pages.** Next improvement is human tightening for flagship pages where the auto-snippet is accurate but not maximally compelling.
2. **MOC pages should be treated as cluster landing pages.** They are intentionally shorter/navigation-heavy, so measure them on internal-link usefulness and SERP snippet clarity, not article word count.
3. **Vietnamese + English query coverage matters.** Keep bilingual aliases where the English concept is commonly searched, e.g. `predictive programming`, `terrain theory`, `CBDC`, `fiat currency`, `Gnosis`.

### Content gaps to consider later

| Gap | Why it matters | Candidate target |
|---|---|---|
| About / editorial identity page | Conventional E-E-A-T and branded trust signal | New `About Red Pill Wiki` or expand `Cách Đọc Red Pill Wiki` |
| Methodology page for symbolic/pattern analysis | Helps controversial pages avoid fake-certainty reading | Expand `Source Discipline` or create `How We Read Patterns` |
| Cluster summaries for AI citation | AI systems like direct definitions and scope boundaries | Add short answer-first sections to MOCs |
| Search-intent bridges | Some notes are deep but not query-shaped | Add `Từ Khóa Cần Hiểu` sections where missing |

## Measurement plan after GSC exists

- Track impressions/clicks/CTR by cluster using URL folder and title regex.
- Identify queries where page ranks but description/title mismatch suppresses CTR.
- Promote pages with impressions but low average position into editorial expansion queue.
- Use GSC query language split to decide whether Vietnamese or English alias should be foregrounded.
